Methodology & Assumptions

Full transparency on how we calculate these numbers.

GPU Cost Assumptions

GPU costs are based on typical cloud on-demand pricing as of 2025. B200 at $6.00/hr ($52,560/yr), H200 at $4.50/hr ($39,420/yr), H100 at $3.50/hr ($30,660/yr), MI300X at $4.00/hr ($35,040/yr), A100 at $2.50/hr ($21,900/yr), L40S at $1.67/hr ($14,600/yr), A10G at $1.25/hr ($10,950/yr). Reserved instance or spot pricing would lower absolute costs but the relative savings ratio stays the same.

Replica & GPU Mapping

GPUs per replica scale with model size: 7B and 8B models fit on 1 GPU, 13B on 1 GPU, 34B needs 2 GPUs (tensor parallel), 70B needs 4 GPUs, and 405B needs 8 GPUs. Throughput per replica scales inversely with model size: 7B handles ~18 req/s, 8B ~16, 13B ~10, 34B ~5, 70B ~3, and 405B ~1 req/s. Replicas include 30% headroom for burst handling.

Cache Savings Factors

Cache-aware routing savings vary by workload. Multi-turn Chat: 40% (shared conversation history). Agentic: 50% (repeated tool schemas and context). RAG: 30% (shared system prompts, varied documents). Code Completion: 35% (shared file context). Batch: 15% (minimal prefix overlap). These are based on published research and production deployments.

TTFT Improvement

Time-to-first-token improvements come from skipping KV-cache computation for cached prefixes. Multi-turn Chat: 4x, Agentic: 5x, RAG: 3.5x, Code: 3x, Batch: 2x. These multipliers apply to the prefill phase only; total end-to-end latency improvement depends on generation length.

CO₂ Calculations

CO₂ estimates use approximately 2.8 metric tons per GPU per year, based on average US data center power usage and carbon intensity. Actual figures vary significantly by region, PUE, and energy mix. This is a directional estimate, not a precise measurement.

What is llm-d?

llm-d is a CNCF Sandbox project for intelligent LLM inference scheduling. Its Endpoint Picker Policy (EPP) integrates with Kubernetes Gateway API to route requests to vLLM pods that already have relevant KV-cache entries, dramatically reducing redundant prefill computation across your serving fleet.
